# Security Policy

## Reporting a Vulnerability

Security is important to ARIA. If you discover a security vulnerability, please report it responsibly.

Please do not open a public GitHub issue for security vulnerabilities.

Instead, contact the maintainers privately with:

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested mitigation or fix (if available)

We will review the report and work on a resolution before public disclosure.

## Scope

Security issues related to the following areas are in scope:

- ARIA backend services
- AI agent workflows
- Casper smart contracts
- On-chain transaction handling
- Authentication and authorization
- External API integrations

## Best Practices

Contributors should:

- Never commit private keys, API keys, or secrets.
- Avoid sharing sensitive deployment information publicly.
- Test smart contract changes on Casper Testnet before deployment.

Thank you for helping keep ARIA secure.
